# Figma UI 文档包说明

## 1. 这套文档现在改成什么了

这套文档不再服务于“学员端一套、工作人员端一套”的拆分式出图方式。

从现在开始，它服务的是一个新的 UI 方向：
- **一个系统**
- **一套账号体系**
- **一套信息架构**
- **手机端和电脑端共用同一产品逻辑**
- **登录时按身份推荐不同首页和入口层级**
- **培训与业务放在同一条系统里，不再割裂**

也就是说，后续给 Figma AI 的所有输入，都要默认站在这个前提上：

> **这是同一个产品在不同设备上的适配，不是两个系统；这是同一个系统在不同身份下的推荐差异，不是两套产品。**

## 2. 使用前提

请始终基于以下前提使用这套文档：
- 当前项目不是单纯的培训页面，而是一套围绕“训练、同步、协同、回流”建立起来的业务系统
- 当前最关键的问题不是页面多少，而是跨角色信息差是否被前置发现、及时同步、持续追踪并最终回流成标准
- 一期范围仍以现有方案为准，不要把老师展示的案例或方法演示误扩成完整系统范围
- 培训是核心模块，但不是独立系统；它必须和销售、设计、审单、工厂、售后、带教、管理联动
- 学员视角和工作人员视角只是**推荐顺序不同、入口层级不同、权限深度不同**，不是内容彼此消失

## 3. 新的设计假设

### 3.1 产品结构假设

这套文档默认采用以下统一产品逻辑：
- 登录时先选本次身份：`学员` 或 `工作人员`
- 身份影响首页推荐、快捷入口、少量专属入口
- 另一部分模块不会因此消失，只是排序或可见层级变化
- 学员满足条件后，可发起“转工作人员申请”
- 审批通过后，账号主身份变为工作人员，但仍保留学习能力与历史记录

### 3.2 双端适配假设

这套文档默认采用以下终端形态：
- **手机端**：轻量任务化界面，底部固定 5 个导航
- **电脑端**：左侧一级导航固定，当前模块展开二级导航，顶部有身份条和全局操作

注意：
- 这不是“移动端做学员、桌面端做工作人员”的硬拆分
- 而是**同一产品在手机和电脑上的不同布局表现**
- Figma AI 输出时，应优先理解“同一个页面的双端适配关系”，而不是画成两套互不相干的页面

## 4. 统一导航规则

### 4.1 手机端固定 5 个一级导航
- `首页`
- `学习`
- `工作台`
- `消息`
- `我的`

其中：
- 学员首页优先展示：学习任务、陪练、考核、成长提醒
- 工作人员首页优先展示：今日待办、待处理、风险提醒、快捷入口
- 但 `学习` 和 `工作台` 两边都保留，只是展示权重不同

### 4.2 电脑端导航规则

电脑端采用：
- 左侧固定 5 个一级导航：`首页`、`学习`、`工作台`、`消息`、`我的`
- 当前一级模块下，在左侧下半区或展开区展示二级导航
- 顶部区域展示：当前身份、切换视角、审批状态、全局搜索、通知入口
- 内容区负责主要任务处理，右侧可配详情抽屉或辅助信息区

## 5. 文件说明

- `01-Figma AI 项目总输入.md`：整套项目的总提示词，先把 Figma AI 拉到“统一双端、一体化系统”的正确方向上
- `02-学员端页面提示词.md`：不是单独学员系统，而是**统一产品内的学员优先视角提示词**
- `03-工作人员端页面提示词.md`：不是单独后台系统，而是**统一产品内的工作人员优先视角提示词**
- `04-页面清单与出图顺序.md`：按统一产品主链路告诉 AI 先画什么、后画什么
- `05-视觉与交互约束.md`：限制视觉风格、配色、状态表达和交互逻辑，避免再画丑或画裂
- `06-Cloud4.6精简提示词.md`：更适合 Cloud 4.6 / Figma AI 对话式协作的短提示词版本

## 6. 建议的 Figma 使用顺序

### 第一步：先建立统一产品骨架
先使用 `01-Figma AI 项目总输入.md`。

目标是让 Figma AI 先理解：
- 这是一个什么系统
- 为什么它不是普通培训 App
- 为什么它不是两个系统
- 统一导航和双端适配应该怎么理解
- 登录、身份、首页推荐、培训与业务联动应该怎么落结构

### 第二步：再按“当前优先视角”细化
- 如果先细化学员常用路径，使用 `02-学员端页面提示词.md`
- 如果先细化工作人员常用路径，使用 `03-工作人员端页面提示词.md`

注意：
- 这两个文件是**同一产品下的不同细化入口**
- 不代表后续要生成两套完全不同的产品风格或导航体系

### 第三步：限制 AI 不要跑偏
在继续细化页面前，把 `05-视觉与交互约束.md` 里的约束补给 AI。

这样可以避免：
- 页面继续被画成两套系统
- 首页变成展示页
- PC 后台变成普通 CMS
- 手机端变成内容社区或营销 App
- 页面出现 `@image:image.png` 这种占位感很强的伪设计结果

### 第四步：按主链路推进出图
参考 `04-页面清单与出图顺序.md`。

建议顺序：
1. 统一骨架与登录 / 身份流转
2. 学习训练主链路
3. 工作台协同主链路
4. 消息、我的、状态页和视觉细化

## 7. 最推荐的实际用法

如果你准备直接开始，我建议按下面这套节奏：

1. 把 `01-Figma AI 项目总输入.md` 整段喂给 Figma AI
2. 让它先出统一产品信息架构、双端导航映射和首页骨架
3. 如果你想先稳住学员常用流程，再喂 `02-学员端页面提示词.md`
4. 如果你想先稳住工作人员工作流，再喂 `03-工作人员端页面提示词.md`
5. 然后用 `05-视觉与交互约束.md` 去统一风格、配色和状态
6. 最后用 `06-Cloud4.6精简提示词.md` 做中途收紧、续写和修正

这样做的好处是：
- 不会一开始就把 AI 拉回“分系统”的旧逻辑
- 更容易把手机和电脑做成同一个产品
- 更容易把培训和业务放在一起设计
- 更适合你后面自己在 Figma 里继续精修
